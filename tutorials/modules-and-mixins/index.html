Chapter 40 – Modules & Mixins Intermediate

Share behavior across classes by grouping methods in modules and mixing them in with `include` or `extend`.

> Adapted from Satish Talim's "Modules & Mixins" lesson. Modules let you group related methods/constants and mix them into classes without multiple inheritance.

include adds instance methods


module Honks
  def honk
    "honk!"
  end
end

class Car
  include Honks
end

puts Car.new.honk  #=> "honk!"

extend adds class-level behavior


module Logging
  def log(message)
    puts "[#{name}] #{message}"
  end
end

class Service
  extend Logging
end

Service.log("Ready")  # class method

Modules as namespaces


module Payments
  class Receipt; end
  end

  Payments::Receipt.new
Modules keep related classes/constants organized and prevent name clashes.
