Chapter 33 – Ruby Exceptions Beginner

Handle errors with `begin`/`rescue`, ensure cleanup, and raise your own exceptions when something goes wrong.

> Adapted from Satish Talim's "Ruby Exceptions" lesson. Ruby uses exceptions to report runtime problems. Use begin/rescue blocks to catch them, ensure to run cleanup, and raise to signal your own errors.

Basic pattern


begin
  risky_operation
rescue ZeroDivisionError => e
  puts "Oops: #{e.message}"
rescue StandardError => e
  puts "Generic error: #{e.class}"
else
  puts "No errors!"
ensure
  puts "Always runs"
end
  • rescue SpecificError => e lets you branch per exception type.
  • else executes only when no exception was raised.
  • ensure runs regardless of success or failure--perfect for closing files or releasing resources.

Raising exceptions

raise (alias fail) triggers an exception: 

raise "Something went wrong"
raise ArgumentError, "Bad argument"
raise ArgumentError.new("Bad argument")
Calling raise with no arguments inside rescue re-raises the current exception.

Legacy example


def divide(x, y)
  raise ArgumentError, "y must not be zero" if y.zero?
  x / y
end

begin
  puts divide(10, 0)
rescue ArgumentError => e
  puts e.message
ensure
  puts "Division attempted"
end

Custom exceptions

Define your own by inheriting from StandardError: 

class ServiceError < StandardError; end

  raise ServiceError, "Remote API unavailable"
